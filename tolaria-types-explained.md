# The Default Types and Why They Exist

Tolaria comes with seven default types. Understanding the type system helps you organize your knowledge effectively.

## The Two- Axis Framework

Every piece of knowledge can be categorized along two dimensions:

|  | One-off | Recurring |
|--|---------|-----------|
| One sitting | Task | Procedure |
| Multiple sittings | Project | Responsibility |

### Tasks — Don't Use the Vault

Tasks that can be completed in one sitting are not meant for Tolaria. Use dedicated task management tools like Todoist, Linear, or Things. Keep your vault for knowledge that has lasting value.

### Projects — One-off, Multiple Sittings

A project is a one-time effort that spans multiple work sessions. It has a defined start and end. Examples: writing a book, launching a product, planning a trip. Projects deserve documentation—you'll return to them over time.

### Responsibilities — Recurring, Multiple Sittings

Ongoing duties that require regular attention. Examples: maintaining a newsletter, managing a team, running a weekly meeting. Responsibilities grow richer over time as you add context and reference.

### Procedures — Recurring, One Sitting

Step-by-step processes you follow regularly. Once documented, a procedure is complete. Examples: weekly review, daily standup format, onboarding checklist. The value is in having a reliable reference.

### Note — The Catch-All

For anything that doesn't fit the above. Links, snippets, ideas, references—anything goes. Note is the fallback type, useful for knowledge that doesn't need structured handling.

## Additional Types

### Topic — Knowledge Aggregator

Topics are long-term containers for related notes. Great for clipping resources from the web, collecting research, or building reference libraries. Topics help you find related notes and see the full picture.

### Person — Lightweight CRM

People you interact with. Attach to projects, events, and responsibilities. Over time, this becomes an automatic CRM—notes about meetings, preferences, context. Works especially well when linked to events.

### Event — Calendar as Notes

Calendar events and meetings become notes. [[luca-rossi]] syncs his calendar and meeting notes (from Fathom/Granola) into event notes, then links events to people → automatic CRM that grows over time.
